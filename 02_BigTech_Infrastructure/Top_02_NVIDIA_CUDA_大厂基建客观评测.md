# 星火计划 (Project Spark) - 核心基础设施冶炼评估
## 2. NVIDIA CUDA

### 1. 核心定位
**定义**: 统治级并行计算架构与GPU底层硬件编程接口设施
**架构简述**: 直接调度底层异构计算阵列，对海量线程池进行微秒级别的任务下派与显存寻址。

### 2. 生态与零信任视阈
**生态边界**: 无 CUDA 不AI；它是所有大模型框架绕不开的深层铁幕，但也意味着企业对 底层硬件调用黑盒几乎没有审计力。
**安全侧写**: 完全掌握设备的DMA/显存控制权，极易导致不同容器/租户之间的超界显存读取（侧信道/越权窃取）。

### 3. Valhalla Matrix 评级
- **静态安全体系评级**: A- (闭源黑盒特权)
- **动态应用拓展评级**: S+ (硬件垄断级性能)

### 5. 深度解剖
#### 5.1 架构亮点
- **统一虚拟内存 (UVM)**：抹平了 CPU 与 GPU 的内存寻址鸿沟，使得海量参数的 LLM 能够平滑地进行 Offload 卸载计算。
- **流处理器极速上下文切换**：支撑了矩阵乘法（GEMM）在成千上万个计算核心上无缝流转。

#### 5.2 隐患暗雷
- GPU 显存残留跨租泄漏：在一个多租户集群中，如果释放 GPU 后未执行强制清零冲洗(Scrubbing)，下一任租户可以直接读出上一任的残余 prompt 与响应。
- 恶意 Shader 提权：精心构造的 CUDA Kernel 可能会造成设备总线挂死或利用驱动漏洞在宿主机提权提升到 Root。

#### 5.3 收容措施 (Containment Strategy)
使用 NVIDIA 官方的 vGPU/MIG 进行严格硬件分区隔离，并在调度侧执行严苛的 GPU 物理清零。对于公有云，永远不要假设裸金属上的 GPU 内存是干净的。

### 6. 实弹推演剧本
- **✅ 容忍境地**: 使用 NVIDIA MIG 进行硬隔离的公有池（推荐度：★★★★☆）
  *阐述*: 将物理卡切分为多个强隔离实例，保障每个部门/框架获得独立的显存与计算单元边界。
  
- **❌ 致命雷区**: 跨业务容器共享单一设备的直接透传（推荐度：★☆☆☆☆）
  *阐述*: 容器仅仅通过 cgroups 隔离，但都可以读写同一张大卡的 `/dev/nvidia0`，极易发生显存越界探查。
